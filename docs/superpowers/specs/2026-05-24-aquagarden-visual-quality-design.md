# 아쿠아가든 비주얼 퀄리티 업그레이드 — 디자인 문서

작성일: 2026-05-24
방향: **B (스타일라이즈드 리치)** — 로우폴리곤·유휴 GPU 0·미니멀 제약을 지키면서 "빛과 분위기"로 체감 퀄리티를 끌어올린다.

## 0. 배경 / 문제

현재 오브제 디자인 퀄리티가 레퍼런스(사진 같은 아쿠아스케이프, `reference_image.png`)와 크게 다르다. 핵심 원인:

- 모든 오브제가 `MeshBasicMaterial`(언릿)이라 `DirectionalLight`/`AmbientLight`가 **아무 일도 하지 않는다** → 입체감·재질감 0
- 물 느낌(뎁스/색감/커스틱/라이트 샤프트) 부재
- 물고기는 아이코사면체+삼각 꼬리로 종 구분 불가, 수초는 단색 삼각 블레이드

제약(requirement.md): 물고기 ~1,000~2,000 트라이앵글 로우폴리곤 **에셋 사용**, 유휴 시 GPU 점유 0에 근접, 수초는 버텍스 셰이더 애니메이션, 투명 창(알파 배경, 스카이박스 없음), 미니멀 구성(시선은 물고기에 집중).

리서치 결론: 위 제약 안에서 **재질을 조명 받게 전환 + 작은 환경맵 IBL + 커스틱 + 물 뎁스 틴트**가 GPU 비용 대비 가장 큰 시각적 향상이며, 이 셋이 체감 퀄리티의 ~80%를 차지한다.

| 항목 | 결정 |
|------|------|

## 2. 아키텍처 / 레이어 (기존 규칙 준수)

- 3D 코드는 `src/renderer/`, OS 제어는 `src/main/`, 공유 타입/상수는 `src/shared/`
- 매직 넘버는 `src/shared/config.ts` 상수로
- 순수 로직(벡터/매핑/UV 애니메이션 계산)은 부수효과(렌더·OS)와 분리해 **TDD**

## 3. 컴포넌트 설계

### 3.1 렌더링 파이프라인 전환 (기반)

- 전 오브제 재질을 **조명 받는 재질**로 전환: 물고기 `MeshStandardMaterial`, 돌/모래는 더 가벼운 `MeshLambertMaterial` 허용
- `scene.environment` 에 **PMREM(`PMREMGenerator.fromScene(RoomEnvironment())`)** 주입 → 저폴리 면이 매끄럽게 모델링된 것처럼 읽힘. 비용은 1회 startup prefilter
- `scene.background` 는 **null 유지** (투명 창 보존). `WebGLRenderer({ alpha:true, premultipliedAlpha:false, antialias:true })`, `setClearColor(0x000000, 0)`
- **밝기 슬라이더 통합**: 기존 `Lighting.setBrightness01` 이 directional/ambient intensity뿐 아니라 `scene.environmentIntensity`(또는 머티리얼 `envMapIntensity`)도 함께 구동 → 밤(어둑한 심해)↔낮(맑고 환함) 디밍이 IBL까지 반영
- **순수함수(TDD)**: `밝기01 → { directional, ambient, envIntensity }` 매핑 (기존 `lightingHelpers` 확장)

### 3.2 물고기 (GLB + 풀링 + 셰이더 벤딩)

- **소싱**: CC0/무료 로우폴리 어류 GLB 후보 수집 → 사용자 승인. 종 구성(안): 군집용 슬림 소형어 2종 + 개체용 중형 2~3종(엔젤/구라미/금붕어류 등). 라이선스/트라이앵글 수(≤~2k) 검증
- **로딩**: `GLTFLoader` 로 종별 1회 로드 → 프로토타입 보관. 인스턴스는 클론(geometry/텍스처 공유)
- **풀링**: `ObjectPool`/`Fish` 리팩터 — 절차적 geometry 대신 종별 GLB 클론 사용. 인스턴스별 색·크기 시드 변주. 개체수 슬라이더는 기존 `spawnPerTick` 비동기 분할 생성 유지(프레임드랍 방지)
- **헤엄 애니메이션**: 정적 GLB + `material.onBeforeCompile` 버텍스 셰이더로 몸통을 진행축 기준 사인파로 휨 + 꼬리 가중. 개체당 추가 비용 0에 근접
- **재질 폴리시**: GLB 머티리얼을 `MeshStandardMaterial` 로 보장 + `onBeforeCompile` 로 **림라이트/프레넬** 주입 → 저폴리도 입체감
- **순수함수(TDD)**: 몸통 벤딩 위상/오프셋, 시드 기반 색·크기 변형, 종 선택 분포

### 3.3 아쿠아스케이프 (수초 + 하드스케이프, 풍성하게)

- **수초**: 삼각 블레이드 → **알파컷아웃 그래스 카드**(교차 quad, 한 장의 잎 알파 텍스처, `alphaTest:0.5` 로 정렬 비용 회피·불투명 패스 유지). 버텍스 셰이더 스웨이(루트 고정, 끝만 흔들림, 높이 가중 `sin(time + worldX*k) * heightFactor`). `InstancedMesh` + 인스턴스별 위상/색 오프셋으로 **2~4종 레이어드**(전경 카펫 / 중경 부쉬 / 배경 키큰 풀)
- **하드스케이프**: 돌 + **유목(driftwood)** 추가, 조명 받는 재질로 음영. 군락 수를 늘리되 모두 하단 배치해 물고기 시야 보존
- **모래**: 단색 → 살짝의 색·노멀 변주 + 커스틱 받는 lit 재질
- **커스틱**: 타일링 커스틱 텍스처(또는 절차적 voronoi/`abs(sin)` GLSL)를 2중 UV 스크롤(서로 다른 속도)로 모래에 일렁임 + 동일 텍스처를 위에서(월드 XZ 투영) 물고기/돌에 투사 → 빛 그물 효과. 영향 표면당 1~2 텍스처 페치
- **순수함수(TDD)**: 커스틱 UV 애니메이션, 그래스/하드스케이프 배치 시드, 스웨이 높이 가중

### 3.4 물 분위기 & 이펙트 (투명창 호환)

- **물 뎁스 틴트**: `THREE.Fog`/`FogExp2` 는 투명창에서 불투명 안개 사각형을 만들고 `ShaderMaterial` 에 무시되므로 **미사용**. 대신 `onBeforeCompile` 로 깊이 기반 (a) 표면색을 틸(teal) 워터 틴트로 lerp, (b) 먼 지오메트리 알파 페이드 → 투명 가장자리 클린
- **라이트 샤프트(god rays)**: 위에서 내려오는 반투명 cone/quad 평면 2~3장, `AdditiveBlending`, 부드러운 수직 그라디언트, UV 천천히 드리프트
- **기포**: 기존 시스템 유지 + 소프트 additive 스프라이트(부드러운 라디얼 알파)로 업그레이드. additive는 정렬 불필요
- **하이라이트 글로우**: 전체화면 블룸(`UnrealBloomPass`)은 투명창에서 프래그먼트 알파를 1로 강제해 배경이 검게 깨짐 → **미사용**. 대신 물고기 눈·커스틱 핫스팟에 additive 글로우 스프라이트로 "가짜 블룸"
- **순수함수(TDD)**: 깊이→틴트/알파 계수 매핑, 샤프트/기포 UV·위치 애니메이션

## 4. 데이터 흐름

밝기 슬라이더(ControlPanel) → IPC → renderer `Lighting.setBrightness01` → directional/ambient/envIntensity 동시 갱신. 개체수 슬라이더 → `FishSchool`/`ObjectPool` grow/shrink → GLB 클론 활성/비활성. 시간 누적(`update(dt)`) → 그래스 스웨이·커스틱·샤프트·기포·물고기 벤딩 셰이더 uniform 갱신. 창 hidden → `RenderLoop` 정지(기존 유지).

## 5. 성능 / 에러 처리

- 추가 효과는 모두 프레임당 저비용(셰이더 ALU + 소수 텍스처 페치). PMREM·GLB·텍스처 로드는 1회 startup
- 유휴 시 렌더 정지 유지. AnimationMixer 미사용으로 개체수 증가에도 CPU 선형 부담 최소
- GLB 로드 실패 시: 해당 종 스킵 또는 폴백 절차적 메시(개발 중 가드), 콘솔 경고

## 6. 테스트 전략

- 순수함수 우선 TDD: 밝기 매핑, 물고기 벤딩/시드 변형/종 분포, 커스틱·스웨이·샤프트·기포 UV/위치 애니메이션, 깊이→틴트/알파 매핑, 배치 시드
- 렌더·OS·GLTF 로드는 순수함수로 분리해 테스트하고, WebGL 부수효과 자체는 단위테스트 제외(기존 패턴)

## 7. 구현 순서 (리서치 권고)

1. **렌더링 파이프라인 전환** (lit 재질 + IBL + 밝기 슬라이더 통합) — 가장 큰 향상
2. **물고기** (GLB 소싱·승인 → 로더·풀링 리팩터 → 셰이더 벤딩 + 림라이트)
3. **커스틱 + 아쿠아스케이프** (그래스 카드, 하드스케이프/유목, 모래, 커스틱 투사)
4. **물 분위기** (뎁스 틴트, 라이트 샤프트, 기포 업그레이드, 글로우 스프라이트)
5. 마무리/튜닝

①~③이 체감 퀄리티의 ~80%를 차지한다.
